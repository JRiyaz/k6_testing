Test Results
-----------
Based on the Autolance performance test results, we confirm that all the functionalities related to Autolance are working as expected.

Additional Observation
--------------------
During testing, we identified a critical memory issue:
- Memory Overconsumption: Some Zope nodes intermittently consumed 5 to 7 GB of memory, despite the Autolance memory limit is set to 750 MB.
- Impact: This has causes server memory utilization to reach 100%.
- Consequence: Other Zope nodes on the same server has crashed with Memory Error.

Configuration to tackle this issue
---------------------------------
To address these memory over-consumption issue, we suggest adjusting the following Autolance configurations:

1. interval:
   - Purpose: Defines how frequently Autolance has to check the memory usage of Zope node.
   - Current default: 360 seconds (6 minutes)
   - Recommended value: Less than 30 seconds
   - Rationale: Test results showed that Zope nodes exceed 2 GB memory usage within the current 6-minute window. Shorter intervals enable to identify memory usage quicly and take action accordingly.

2. hard_restart_after:
   - Purpose: Defines the maximum wait time before restarting a Zope node.
   - Current default: 0 (zero) - waits indefinitely for pending tasks/requests to complete
   - Recommended value: Less than 60 seconds
   - Behavior:
     * Value = 0: Graceful restart (waits for all pending tasks/requests to complete)
     * Value > 0: Hard restart (forcefully restart after specified duration)
   - Rationale: The current default value allows nodes to run indefinitely while consuming excessive memory. Setting a specific timeout ensures regular memory cleanup.
